# PRD — WarungDigital: Sistem Kasir & Inventaris Berbasis Google Workspace

**Versi:** 1.0
**Status:** Draf
**Platform Target:** Google Apps Script (V8 Runtime) + Google Sheets
**Akses:** Web App (responsif, mobile-first)

---

## 1. Ringkasan Eksekutif

WarungDigital adalah aplikasi kasir dan manajemen inventaris berbasis web yang berjalan di atas infrastruktur Google Workspace gratis. Target pengguna adalah pemilik warung/toko kecil di Indonesia yang punya HP Android/kirim dan koneksi internet minimal. Aplikasi ini menggunakan **Google Sheets sebagai database**, **Google Apps Script sebagai backend API**, dan **HTML/CSS/JS sebagai frontend** — tanpa biaya server, tanpa aplikasi tambahan, cukup browser.

## 4. Backend (Code.gs) — Spesifikasi Teknis

### 4.1 Entry Point

```javascript
// Melayani UI: file HTML di-render sesuai halaman
function doGet(e) {
  const page = e.parameter.page || 'login';
  const html = HtmlService.createHtmlOutputFromFile(page)
      .setTitle('WarungDigital')
      .setXFrameOptionsMode(HtmlService.XFrameOptionsMode.ALLOWALL)
      .addMetaTag('viewport', 'width=device-width, initial-scale=1');
  return html;
}
```

Halaman yang akan disediakan:
- `login.html` — form login
- `dashboard.html` — ringkasan, akses cepat
- `kasir.html` — halaman transaksi (scan barcode, pilih produk)
- `produk.html` — manajemen produk (CRUD)
- `laporan.html` — laporan penjualan & pengeluaran
- `pengeluaran.html` — catat pengeluaran
- `pengaturan.html` — profil toko, WA, user

### 4.3 Optimasi Performa untuk Ribuan Baris

Teknik yang digunakan:

1. **Batch GET**: Gunakan `sheet.getDataRange().getValues()` sekali, filter di JavaScript daripada panggil Sheets API berulang
2. **Caching**: Gunakan `CacheService.getScriptCache()` untuk lookup cepat (produk, kategori) dengan TTL 30 menit
3. **Batch UPDATE**: Gunakan `range.setValues()` untuk update massal (bukan per cell)
4. **TextFinder**: Gunakan `sheet.createTextFinder()` untuk lookup cepat barcode/produk
5. **LockService**: Prevent race condition saat dua transaksi concurrent
6. **Batch import**: Proses 100 baris per panggilan `setValues()` untuk menghindari 6 menit execution limit

### 4.4 Autentikasi

Mekanisme **session-based sederhana** tanpa OAuth:

1. **Login**: User input username + password
2. **Verifikasi**: Backend cek `User` sheet, hash password dengan `Utilities.computeDigest(Utilities.DigestAlgorithm.SHA_256, ...)`
3. **Session token**: Generate UUID, simpan di `PropertiesService.getUserProperties()` + `Sesi` sheet
4. **Set cookie**: Kirim token ke client via `google.script.run` callback, simpan di `sessionStorage`
5. **Validasi**: Setiap panggilan CRUD bawa token, divalidasi di server
6. **Expiry**: Token expired 12 jam

```javascript
function doLogin(username, password) {
  const hash = sha256(password);
  const userSheet = getSheet('User');
  // lookup + verifikasi
  const token = generateUUID();
  saveSession(token, username);
  return { success: true, token, namaToko, namaUser };
}
```

### 4.5 Notifikasi WhatsApp

Menggunakan **CallMeBot API** (gratis, personal use):

```javascript
function sendWANotif(phone, message) {
  const encoded = encodeURIComponent(message);
  const apikey = PropertiesService.getScriptProperties().getProperty('WA_APIKEY');
  const url = `https://api.callmebot.com/whatsapp.php?phone=${phone}&text=${encoded}&apikey=${apikey}`;
  UrlFetchApp.fetch(url, { muteHttpExceptions: true });
}
```

Trigger notifikasi:
- **Stok menipis** (< Stok_Min): dikirim otomatis setelah transaksi
- **Struk pembelian**: opsional, dikirim ke WA pembeli (input manual)
- **Rekap harian**: trigger cron (opsional)

### 4.6 Generate QRIS

Gunakan library client-side **qrcodejs** via CDN:

```html
<script src="https://cdn.jsdelivr.net/npm/qrcodejs@1.0.0/qrcode.min.js"></script>
```

QRIS statis: QR code diisi format `QRIS://${nmid}?amount=${total}` — ditampilkan di layar setelah transaksi QRIS dipilih.

### 5.3 Scan Barcode

Menggunakan **html5-qrcode** (library 100% client-side, tanpa backend):

```html
<script src="https://unpkg.com/html5-qrcode"></script>
```

```javascript
function startScanner() {
  const scanner = new Html5Qrcode("reader");
  scanner.start(
    { facingMode: "environment" },  // kamera belakang
    {
      fps: 10,
      qrbox: { width: 250, height: 150 }
    },
    (decodedText) => {
      // decodedText = barcode terdeteksi
      scanner.stop();
      cariProdukByBarcode(decodedText);
    }
  );
}
```

Format barcode yang didukung: EAN-13, EAN-8, CODE-128, CODE-39, UPC-A, UPC-E, QR Code, DataMatrix, PDF417.

### 5.6 Flow Halaman Kasir

1. User buka halaman Kasir
2. Input produk: **scan barcode** (kamera aktif) ATAU **pilih dari daftar** (search + filter kategori)
3. Setelah produk terpilih: otomatis ambil harga jual, input quantity
4. Tambah produk ke keranjang (daftar di bawah)
5. Jika selesai: tekan **"Hitung Total"**
6. Pilih metode bayar: `[Tunai]` / `[QRIS]`
7. Jika Tunai: input nominal bayar, hitung otomatis kembalian
8. Jika QRIS: generate QR code dari total, tunjukkan ke pembeli
9. Tekan **"Simpan Transaksi"** → simpan ke Sheets, kurangi stok, kirim WA notifikasi
10. Tampilkan struk (bisa di-screen capture)

## 6. Integrasi Pihak Ketiga

### 6.1 WhatsApp Notification (CallMeBot)

- **Biaya**: Gratis (personal use, max ~50 pesan/hari)
- **Setup**: Aktivasi sekali (kirim "I allow callmebot to send me messages" ke +34644204756)
- **API Key**: Disimpan di `PropertiesService.getScriptProperties()`
- **Cara panggil dari GAS**: `UrlFetchApp.fetch('https://api.callmebot.com/whatsapp.php?...')`

**Catatan**: CallMeBot bisa lambat/heboh. Alternatif berbayar: Twilio ($0.005/pesan).

### 6.2 QRIS (QR Code)

- **Library**: `qrcodejs` (CDN) — 100% client-side
- **Format**: Generate QR dari string format QRIS standard
- **Tampilan**: Ditampilkan di halaman kasir setelah pilih QRIS, ukuran minimal 250x250px
- **Tidak perlu API pihak ketiga**: QR code digenerate di browser langsung

### 6.3 Optional: Google Charts

Untuk laporan grafik di `laporan.html`:

```html
<script src="https://www.gstatic.com/charts/loader.js"></script>
```

Gratis, tanpa API key. Bisa buat bar chart penjualan harian, pie chart kategori.

## 7. Keamanan

| Aspek | Implementasi |
|-------|-------------|
| Password | Hash SHA-256, tidak ada plaintext |
| Session | Token UUID, validasi tiap request, expiry 12 jam |
| Input | Sanitasi di server (hindari injection) |
| Sheet protection | Tab sensitif (User) bisa diproteksi manual |
| Deploy | Set "Execute as: Me" + "Who has access: Anyone" |

| XSS | Hindari `innerHTML` — gunakan `textContent` untuk data user |

**Catatan**: Karena menggunakan akun Google pribadi, data aman se-aman akun Google pemilik.

## 10. Risiko & Mitigasi

| Risiko | Dampak | Mitigasi |
|--------|--------|----------|
| GAS quota 90 menit/hari | Gagal transaksi jika overload | Limit usage, cache berat, batch write |
| CallMeBot down/no response | WA notif gagal | Log error, fallback ke Gmail, kasih notif in-app |
| Koneksi internet lambat | UI lemot | Gunakan cache (CacheService), UI minimalis |
| HP kentang (RAM kecil) | JS lag | Hindari library berat, bundle kecil |
| Token session hilang | User harus login lagi | Simpan token di sessionStorage |
| Dua user transaksi bersamaan | Data tabrakan (`race condition`) | LockService untuk write critical |
| Google ubah quota/terms | Service terhenti | Monitor email Google, siapkan migrasi |
